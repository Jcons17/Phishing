Phishing¶

Phishing es un trmino informtico que distingue a un conjunto de tcnicas que persiguen el engao a una vctima ganndose su confianza hacindose pasar por una persona, empresa o servicio de confianza (suplantacin de identidad de tercero de confianza), para manipularla y hacer que realice acciones que no debera realizar (por ejemplo revelar informacin confidencial o hacer click en un enlace).

Técnicas de phishing¶

-Phishing general, phishing tradicional, Bulk Phishing o Spray and pray. Consiste en la emisin masiva de correos electrnicos a usuarios. Estos correos suplantan a entidades de confianza (ejemplo bancos) y persiguen el engao del usuario y la consecucin de informacin. Por ejemplo en el mensaje se incluyen enlaces a dominios maliciosos. Para camuflar estos enlaces es habitual que el texto del enlace sea la URL correcta, pero el enlace en s apunte al sitio malicioso.

-Vishing. Es similar al phishing tradicional pero el engao se produce a travs de una llamada telefnica. El trmino deriva de la unin de dos palabras en ingls: 'voice' y 'phishing'. Un ejemplo tpico de uso de esta tcnica es cuando un ciberdelincuente ha robado ya informacin confidencial a travs de un ataque de phising, pero necesita la clave SMS o token digital para realizar y validar una operacin. Es en ese momento el ciberdelincuente llama por telfono al cliente identificndose como personal del banco y, con mensajes particularmente alarmistas, intenta de que el cliente revele el nmero de su clave SMS o token digital, que son los necesarios para autorizar la transaccin.

-Smishing. Es similar al phishing tradicional pero el engao se produce a travs mensajes de texto ya sean por SMS o mensajera instantnea (como WhatsApp). Un ejemplo tpico de esta tcnica es cuando el cliente recibe un mensaje de texto, donde el emisor se hace pasar por el banco, y le informan que se ha realizado una compra sospechosa con su tarjeta de crdito. A su vez, el texto solicita que se comunique con la banca por telfono de la entidad financiera y le brinda un nmero falso. El cliente devuelve la llamada y es ah cuando el ciberdelincuente, hacindose pasar por el banco, solicita informacin confidencial para supuestamente cancelar la compra. En una variante de esta modalidad el mensaje tambin podra incluir un enlace a una web fraudulenta para solicitar informacin sensible.12

-URL Phishing. Se trata de engaar al usuario haciendo que una URL de un sitio malicioso parezca la de un sitio confiable. A ellas a veces se accede de forma inadvertida al escribir nombres de dominio mal escritos que estn muy cerca del dominio legtimo, o siguiendo un enlace malicioso que parece correcto, o por engaos al usar caracteres unicode parecidos difcilmente detectables, especialmente en dispositivos mviles, con pantallas ms pequeas y generalmente una resolucin de pantalla inferior. Por ejemplo caracteres latinos con un punto bajo o la letra griega micron, "". . Otra forma de disfrazar enlaces es utilizar direcciones que contengan el carcter arroba: @, para posteriormente preguntar el nombre de usuario y contrasea (contrario a los). Por ejemplo, el enlace $http://www.google.com@members.tripod.com/$ puede engaar a un observador casual y hacerlo creer que el enlace va a abrir en la pgina de $www.google.com$, cuando realmente el enlace enva al navegador a la pgina de members.tripod.com (y al intentar entrar con el nombre de usuario de www.google.com, si no existe tal usuario, la pgina abrir normalmente). Este mtodo ha sido erradicado desde entonces en los navegadores de Mozilla e Internet Explorer.

    model.add(tf.keras.layers.Dense(8, input_dim=8,activation='sigmoid'))

    # Output layer
    model.add(tf.keras.layers.Dense(1, activation='sigmoid'))

    # Compile a model
    model.compile(loss='binary_crossentropy', 
                  optimizer=tf.keras.optimizers.SGD(learning_rate, momentum),
                  metrics=['accuracy'])
    return model

Conclusiones.¶

  • Podemos ver que la red es muy buena para precisin de datos de entrenamiento y ensayo.

  • No se presenta Overfitting, ya que la diferencia entre la precisin de los datos de entrenamiento y ensayo es del 0.05%.

  • Solo se necesitaron 94 iteraciones para que la red detuviera el proceso de aprendizaje. Si aumentamos el patiente en el parametro EarlyStopping en el callback.

  • Si quitamos los callback y dejamos que corran las 1000 iteraciones la presicin maxima seria de 93.86%.

  • Esta sistema puede ser eficiente para detectar URL-phishing, pero tambien podria tener algunos fallos con otros tipos de phishing. Incluso con URL-phishing, ya que podran modificar una URL modificada y cambiar con un caracter desconocido que la red no detecte.
